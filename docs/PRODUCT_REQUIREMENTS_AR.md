# وثيقة المتطلبات الرسمية للمنصة التعليمية

## 1. تعريف المشروع
المنصة هي نظام تعليم ذكي واحترافي يهدف إلى تعليم الطالب، قياس مستواه، تحليل مواطن الضعف، ثم إعادة توجيهه نحو الإتقان.

المنصة لا تعمل فقط كموقع يعرض كورسات، بل كنظام متكامل يعتمد على الدورة التعليمية والمهارة والاختبار والتحليل والتوصية.

## 2. الرؤية الأساسية
المنصة تعمل وفق دورة:

**تشخيص -> تحليل -> علاج -> إتقان**

بمعنى:
- الطالب يتعلم من خلال دورة أو مهارة أو اختبار مباشر.
- النظام يجمع النتائج ويربط كل سؤال بمهارة.
- يتم تحليل مستوى الطالب على مستوى المهارات.
- يتم اقتراح علاج مناسب: فيديو، اختبار، أو مسار ذكي.
- يستمر الطالب حتى يصل إلى إتقان المهارة.

## 3. المبادئ العامة للتنفيذ
- الحفاظ على الشكل الحالي للواجهة قدر الإمكان.
- الحفاظ على القوائم الديناميكية الموجودة الآن.
- كل الإضافات تكون “من الداخل” وليس تغييرًا بصريًا جذريًا.
- دعم التوسع المستقبلي: إضافة مسارات، مواد، مهارات، مدارس، ومجموعات دون الحاجة لإعادة البرمجة.

## 4. مكونات المنصة الأساسية

### 4.1 نظام الدورات (LMS)
نظام دورات احترافي يشمل:
- إنشاء الدورات
- إنشاء الوحدات
- إنشاء الدروس
- ربط الفيديوهات بالدروس
- إضافة اختبارات داخل الدورة
- ربط الدورة بالمهارات
- تتبع تقدم الطالب داخل الدورة

### 4.2 نظام المهارات (Core System)
المهارة هي الوحدة الأساسية داخل النظام.

كل من:
- السؤال
- الفيديو
- الاختبار
- الدرس

يمكن ربطه بمهارة أو أكثر.

الهدف من ذلك:
- تحليل أداء الطالب على مستوى المهارات
- بناء تقارير مهارية
- إنتاج توصيات دقيقة
- دعم التعلم التكيفي

### 4.3 رحلة الطالب (Student Journey)
يدخل الطالب للمنصة بعدة طرق:
- التسجيل في دورة
- اختيار مهارة مباشرة
- دخول فيديو مرتبط بمهارة
- دخول اختبار مباشر

أثناء الاختبار:
- إضافة السؤال إلى المفضلة
- إضافة السؤال إلى المراجعة لاحقًا

بعد الاختبار:
- عرض الدرجة
- تحليل المهارات
- احتساب أداء كل مهارة بناءً على الأسئلة المرتبطة بها

مراجعة الاختبار:
- عرض الإجابات الصحيحة والخاطئة
- شرح نصي أو فيديو لكل سؤال
- إعادة الاختبار
- اقتراح اختبار جديد أو محتوى علاجي
- إمكانية أن ينشئ الطالب اختبارًا لنفسه حسب الفلترة المتاحة

## 5. لوحة الطالب
يجب أن تشمل على الأقل:

### 5.1 الملف الشخصي
- بيانات الطالب
- المدرسة أو المجموعة
- المرحلة أو التصنيف
- الاشتراك

### 5.2 دوراتي
- جميع الدورات المسجل فيها
- نسبة التقدم
- آخر درس تمت مشاهدته

### 5.3 اختباراتي
- كل المحاولات السابقة
- تصنيف حسب المسارات أو المواد
- الدخول إلى تفاصيل كل محاولة

### 5.4 المفضلة
- الأسئلة المفضلة
- الأسئلة الصعبة
- الأسئلة التي أخطأ فيها
- أسئلة المراجعة لاحقًا

### 5.5 تقارير المهارات
- مهارات ضعيفة
- مهارات متوسطة
- مهارات قوية
- اقتراح علاجي لكل مهارة
- هدف الوصول إلى الإتقان

### 5.6 التوصيات الذكية
- فيديوهات مقترحة
- اختبارات مقترحة
- مسار ذكي علاجي
- إعادة توجيه الطالب بناءً على الأداء

### 5.7 خطتي الدراسية
- خطة دراسية مرتبطة بدورة أو هدف
- توزيع زمني
- تتبع الالتزام

### 5.8 واجباتي
- عرض الواجبات
- المهام
- حالة التنفيذ

### 5.9 سؤال وجواب
- استفسارات متعلقة بالدورات
- تواصل أكاديمي
- رسائل أو استفسارات عامة

### 5.10 طلباتي ومشترياتي
- الدورات المشتراة
- الباقات
- سجل الطلبات

### 5.11 شهاداتي
- عرض الشهادات
- تحميلها

### 5.12 مجموعتي
- المجموعة أو المدرسة
- الأنشطة المشتركة
- ما يخص مشرف المجموعة

## 6. لوحة الإدارة
لوحة احترافية تعتمد على Sidebar Menu وتشمل:

### 6.1 المحتوى التعليمي
- الدورات
- بنك الدروس
- بنك الفيديوهات
- بنك الأسئلة
- الاختبارات
- المكتبة

### 6.2 التصنيفات الهرمية (Hierarchical Taxonomy)
الهيكل الشجري المطلوب:
- مادة رئيسية
- مادة فرعية
- قسم
- مهارة
- مهارة فرعية

ويستخدم هذا الهيكل في:
- الأسئلة
- الفيديوهات
- الدروس
- الاختبارات
- الدورات

### 6.3 بنك الأسئلة
صفحة موحدة تشمل:
- عرض كل الأسئلة
- فلترة حسب المادة والقسم والمهارة
- إنشاء سؤال
- تعديل سؤال
- محرر موحد للأسئلة
- ربط السؤال بمهارة أو أكثر

### 6.4 بنك الفيديوهات
صفحة موحدة مثل بنك الأسئلة:
- فلترة حسب المادة والمهارة
- كل فيديو مرتبط بمهارة
- إمكانية ربط الفيديو بسؤال أو درس

### 6.5 الاختبارات
أنواع الاختبارات:
- اختبارات ساهر
- اختبارات مركزية
- اختبارات الدورات
- بنوك تدريب

مع محرر اختبارات موحد.

### 6.6 المجموعات والمدارس
- إنشاء مدارس
- إنشاء مجموعات
- ربط طلاب
- إضافة مشرفين
- تقارير خاصة بكل مدرسة

### 6.7 العروض والاشتراكات
- باقات
- كوبونات
- عضويات
- صلاحيات حسب نوع الاشتراك

### 6.8 المستخدمون
الأدوار:
- مدير
- مدرب
- طالب
- مشرف
- ولي أمر

### 6.9 التقارير
- تقارير الطلاب
- تقارير المستخدمين
- تقارير المدارس
- تقارير مالية

### 6.10 الإعدادات
- إعدادات عامة
- بوابات الدفع
- الربط مع الخدمات الخارجية

## 7. الأدوار في النظام

### 7.1 الطالب
- يتعلم
- يختبر
- يتابع الأداء

### 7.2 المدرب
- يضيف دورات
- يضيف فيديوهات
- ينشئ اختبارات
- يضيف محتوى أكاديمي

### 7.3 المدير
- يدير كل شيء
- يراقب الأداء
- يتحكم في النظام
- يرى الإحصاءات الكاملة

### 7.4 ولي الأمر
- متابعة الأبناء
- عرض النتائج
- متابعة الأداء

### 7.5 المشرف
- متابعة طلاب مدرسة أو مجموعة معينة
- الاطلاع على تقاريرهم

## 8. النظام الذكي (Adaptive Learning)
يعتمد النظام الذكي على:
- نتائج الاختبارات
- تحليل المهارات
- اكتشاف نقاط الضعف
- تتبع التقدم مع الزمن

ويقوم بـ:
- اقتراح فيديوهات
- اقتراح اختبارات
- إعادة توجيه الطالب
- بناء مسار علاجي

## 9. اقتراحات هندسية إضافية
هذه إضافات موصى بها وتناسب المشروع بدون تغيير شكله العام:

### 9.1 سجل نشاط كامل
- من دخل
- من أنشأ
- من عدل
- من حذف

### 9.2 نظام نشر تدريجي
- مسودة
- منشور
- مجدول

### 9.3 نظام موافقات
- اعتماد المحتوى قبل نشره

### 9.4 إشعارات
- داخل المنصة
- بريد
- واتساب

### 9.5 مزامنة فورية
- باستخدام Socket.IO
- ظهور التغييرات للطلاب فور نشرها

### 9.6 قابلية التوسع
- دعم تطبيق جوال مستقبلًا
- دعم تعدد المدارس
- دعم مزيد من المسارات

## 11. معيار النجاح
يُعتبر المشروع ناجحًا عندما:
- يظل الشكل الحالي محفوظًا
- تصبح البيانات حقيقية وليست وهمية
- يستطيع المدير إنشاء المسارات والمواد والدروس والاختبارات من اللوحة
- يرى الطالب كل ذلك بشكل حي وصحيح
- يتم تحليل الأداء على مستوى المهارات
- يتم اقتراح علاج أو إعادة توجيه للطالب
- تصبح المنصة قابلة للتسليم والتشغيل الحقيقي
