# حالة التطوير الحالية

آخر تحديث: 4 مايو 2026

هذا الملف هو مرجع سريع قبل كل دفعة تطوير. الهدف أن نعرف ما تم، وما تبقى، وأي جزء يجب إغلاقه بالكامل بدل الرجوع إليه مرة أخرى.

## النتيجة المختصرة

- التشغيل والربط: مكتمل بدرجة عالية.
- قاعدة البيانات: MongoDB Atlas هي مصدر البيانات الحالي.
- الواجهة: Vercel للواجهة، Render للـ API، GitHub للكود.
- فحص التشغيل الأخير: 70 فحصًا ناجحًا من 70 عبر `npm run smoke:operational`.
- فحص الواجهة: المسارات الأساسية تعمل، ويتم تأكيد Vercel بعد كل رفع جديد.

## ما تم إغلاقه فعليًا

- تشغيل Vercel -> Render -> MongoDB Atlas.
- تسجيل الدخول لكل الأدوار الأساسية.
- حفظ البيانات في قاعدة حقيقية.
- جاهزية مساحات تعلم رئيسية: القدرات الكمي، القدرات اللفظي، والتحصيلي رياضيات.
- إخفاء المحتوى المعلق عن الطالب والزائر.
- ربط الدروس والاختبارات والملفات بالمسار والمادة والمهارات.
- فحص أن الدروس المرتبطة بالموضوعات المنشورة لها وسائط قابلة للتشغيل.
- تحسين نتيجة الاختبار والتوصيات.
- تحسين تجربة أداء الاختبار: زر مراجعة أوضح، خريطة أسئلة، نظام ليلي، ومنع اختفاء النتيجة قبل الحفظ.
- تبسيط تقرير ولي الأمر: ملخص قابل للنسخ والمشاركة، وخطوات متابعة أسبوعية واضحة بدون ازدحام.
- تبسيط تقرير الطالب: مهارة واحدة يبدأ بها، ثم فتح شرح، تدريب قصير، وإعادة قياس بدل ازدحام الأرقام.
- إضافة بطاقة رحلة مصغرة داخل نافذة الموضوع: الشرح، التدريب، ثم ملف دعم إن وجد.
- إضافة مسار واضح بعد الاختبار: مراجعة الحلول، فهم المهارة، تدريب، ثم متابعة التقرير.
- ربط الخطة الذكية بمتابعة أسبوعية بعد آخر نتيجة: نتيجة -> شرح -> تدريب -> تقرير.
- فصل التدريب عن الاختبارات المحاكية في مركز الاختبارات ومساحة التعلم.
- تأكيد أن إدارة التدريب والاختبارات المحاكية داخل مساحة المادة تستدعي نفس عناصر مركز الاختبارات، مع ربط المسار تلقائيًا عند إنشاء اختبار من داخل المادة.
- إضافة لوحة جاهزية أعلى مساحة عمل المادة تعرض إجمالي المحتوى، الظاهر للطالب، وما يحتاج مراجعة، مع انتقال مباشر لكل إدارة.
- إضافة شارة جاهزية داخل صفوف الدورات والدروس والمكتبة، مع زر تجهيز سريع للطالب عند الحاجة.
- إضافة شارة جاهزية داخل موضوعات التأسيس، مع زر يفتح الموضوع ويجهز الدروس والتدريبات المرتبطة للطالب.
- إضافة زر تجهيز العرض داخل التدريب والاختبارات المحاكية لضبط النشر والاعتماد ومكان الظهور من نفس مركز الاختبارات.
- إضافة معاينة شكل الاختيارات داخل منشئ الاختبار وضبط فلترة المهارات حسب أسئلة الاختبار الفعلية.
- إضافة أدوات إدارية سريعة لتثبيت تصنيف الاختبارات القديمة وتحديد ظهور كل اختبار: تدريب فقط، محاكي فقط، أو الاثنين.
- تحسين المدفوعات اليدوية وطلبات الدفع.
- تنظيف واجهات طالب ظهرت فيها نصوص مشوهة.
- تحويل المدونة إلى مركز محتوى نظيف مبني على الدروس النصية أو الدورات المنشورة.
- إضافة تصدير نسخة احتياطية لمسار واحد من لوحة النسخ، تشمل المسار ومواده ومهاراته وموضوعاته ودروسه وأسئلته واختباراته ودوراته ومكتبته.
- إضافة تبويب ربط ومتابعة داخل إدارة المدرسة: قالب Excel، رفع ملف ربط، ربط أولياء الأمور والمشرفين الموجودين بالطلاب والفصول، وتقرير نواقص قابل للتصدير.
- تطوير تبويب الربط ليقدر اختياريا إنشاء حسابات أولياء الأمور والمشرفين الناقصة من ملف Excel، ثم تصدير ملف تسليم يحتوي الحسابات الجديدة وكلمات المرور المؤقتة.
- إضافة تقرير فصل قابل للتصدير من كل فصل، مع الطلاب والمشرفين والدورات وطلاب الفصل بلا ولي أمر.
- إضافة تقرير باقات وأكواد للمدرسة، ومنع توليد أكواد على باقة موقوفة، وجعل إيقاف الباقة يحفظ إعداداتها بدل الاعتماد على الحذف.

## المتبقي حسب الأولوية

1. رحلة الطالب داخل مساحة التعلم
   - إغلاق تجربة: مسار -> مادة -> موضوع -> درس -> اختبار -> نتيجة -> توصية. تم تعزيز خطوة الموضوع وخطوة ما بعد الاختبار.
   - منع أي بدائل فيديو مربكة عندما يوجد درس مباشر.
   - اختبار الجوال والتابلت واللاب توب.

2. التقارير العلاجية
   - جعل صفحة التقارير مدخلًا واضحًا: أضعف مهارة، شرح مقترح، تدريب مقترح، اختبار قياس. تم تنفيذها للطالب بصيغة مختصرة.
   - تحسين الربط الحي بين نتيجة الاختبار والخطة الذكية. تم تنفيذ متابعة أسبوعية مبسطة، والمتبقي تعميقها بإشعارات ومهام قابلة للتعليم.
   - إبقاء تفاصيل المشرف والمدير في نطاقهم فقط.

3. إدارة مساحة التعلم
   - تثبيت أن النشر يتم من مساحة التعلم لا من المستودعات مباشرة. تم توضيح أن التدريب والمحاكي يستدعيان من مركز الاختبارات نفسه.
   - توضيح حالة كل عنصر: منشور، مخفي، مدفوع، مجاني، يحتاج مراجعة. تم تنفيذ ملخص المادة وشارات الصفوف للدورات والدروس والمكتبة وموضوعات التأسيس والتدريب، والمتبقي توسيع نفس النمط على كل عناصر الإدارة المتقدمة.

4. المدارس والباقات
   - تم تنفيذ ربط جماعي للحسابات الموجودة وإنشاء الحسابات الناقصة اختياريا مع ملف تسليم آمن.
   - تم تنفيذ تقرير ربط ومتابعة للمدرسة، وتقرير فصل قابل للتصدير، وتقرير باقات وأكواد.
   - تم تحسين تفعيل وإيقاف باقات المدارس بدون حذف، والمتبقي لاحقا واجهة طباعة مباشرة من المتصفح بدل Excel فقط.

5. المدفوعات والباقات العامة
   - ربط أوضح بين الباقة وما تفتحه داخل المسار والمادة.
   - بوابة دفع إلكترونية لاحقًا إذا تم اختيار مزود.

6. الذكاء الاصطناعي
   - استخدامه كمساعد داخل التقارير والخطة العلاجية.
   - إبقاء المفاتيح داخل السيرفر فقط.
   - دعم ترتيب مزودات متعدد: Gemini وOpenRouter وQwen وDeepSeek وOpenAI ثم fallback آمن.
   - إظهار حالة الرد للطالب: مخصص حسب الأداء أو احتياطي آمن.

7. الجودة والتوثيق
   - تحديث هذا الملف بعد كل دفعة.
   - تشغيل TypeScript و Build و Smoke API قبل الرفع.
   - تشغيل فحص Vercel بعد الرفع للتأكد أن النسخة الجديدة ظهرت.
   - تم دعم نسخة كاملة للمنصة ونسخة مسار منفصل، والمتبقي لاحقًا ربط خارجي اختياري مثل Google Drive أو مزود تخزين.
