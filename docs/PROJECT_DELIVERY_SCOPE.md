# وثيقة نطاق التنفيذ والتسليم

## الهدف
تحويل المشروع الحالي من واجهة تجريبية قوية إلى منصة تعليمية حقيقية تعمل بقاعدة بيانات فعلية وتزامن حي وصلاحيات صحيحة، مع الحفاظ على الشكل الحالي والهوية البصرية والديناميكية الموجودة في الواجهة.

## التزام التنفيذ
- لا يتم تغيير فكرة المنصة.
- لا يتم تغيير توزيع الصفحات أو مسارات الاستخدام الأساسية إلا إذا كان ذلك لإصلاح كسر واضح.
- تظل إدارة المسارات والمواد والقوائم ديناميكية كما هي الآن، لكن تصبح مرتبطة بقاعدة بيانات حقيقية.
- أي تطوير جديد يكون "إضافة فوق الموجود" وليس إعادة بناء شكل الموقع من الصفر.

## نطاق المرحلة الحالية
### 1. البنية الخلفية
- إنشاء Backend مستقل بـ `Node.js + Express + MongoDB + Socket.IO`.
- تجهيز نماذج البيانات الأساسية المطابقة لهيكل المشروع الحالي.
- تجهيز نظام صلاحيات أولي للأدوار: مدير، معلم، مشرف، طالب، ولي أمر.

### 2. قاعدة البيانات
- نقل المفاهيم الحالية إلى MongoDB:
  - المستخدمون
  - المسارات
  - المراحل
  - المواد
  - المواضيع
  - الدروس
  - الدورات
  - الأسئلة
  - الاختبارات
  - النتائج
  - النشاطات
  - المجموعات

### 3. الربط مع الواجهة الحالية
- إنشاء طبقة `API` وطبقة `adapter` حتى تظل الواجهة الحالية قابلة للعمل بدون تغيير شكلي.
- جعل الربط التدريجي ممكنًا صفحة صفحة بدون كسر تجربة المستخدم.

### 4. التزامن الحقيقي
- تجهيز قناة أحداث فورية لعمليات مثل:
  - تحديث المحتوى
  - نشر اختبار
  - تحديث الدورات
  - تغييرات الإدارة

## مراحل التنفيذ المقترحة
### المرحلة الأولى
- تأسيس Backend.
- تأسيس MongoDB models.
- Health check.
- Auth مبدئي.
- API للكيانات الأساسية.

### المرحلة الثانية
- ربط لوحة الإدارة الحالية.
- ربط المسارات والمواد والدروس والاختبارات.
- نقل الصفحات المعتمدة على `mockData` إلى الربط الحقيقي.

### المرحلة الثالثة
- التزامن الحقيقي.
- صلاحيات مفصلة.
- التقارير.
- النشر والإطلاق.

## المطلوب من صاحب المشروع
- رابط MongoDB Atlas أو الموافقة على إنشائه.
- حساب الاستضافة أو الجهة المفضلة للنشر.
- اسم الدومين إن وجد.
- بيانات الدخول الرسمية التي تريد اعتمادها للإدارة.
- لاحقًا: وسائل الدفع أو واتساب أو البريد إذا رغبت في تفعيلها.

## مبدأ العمل
أي جزء من الواجهة يظهر للمستخدم الآن ويعمل بصريًا، يتم الحفاظ عليه قدر الإمكان، بينما يتم استبدال الربط الداخلي والبيانات الوهمية بطبقة تشغيل حقيقية.
