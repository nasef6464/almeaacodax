# نسخ واسترجاع محتوى التعلم

هذا الملف يشرح طريقة حفظ نسخة احتياطية من محتوى التعلم الموجود في MongoDB Atlas.

## ماذا يشمل النسخ؟

يشمل النسخ عناصر التشغيل التعليمية الأساسية:

- المسارات والمراحل والمواد والأقسام والمهارات
- الموضوعات والدروس والاختبارات والأسئلة
- الدورات والباقات وملفات المكتبة
- إعدادات الصفحة الرئيسية

لا يشمل النسخ بيانات المستخدمين أو كلمات المرور أو طلبات الدفع، حتى لا تختلط النسخ التعليمية مع بيانات حساسة.

## إنشاء نسخة احتياطية

من جذر المشروع:

```bash
npm run backup:learning
```

سينشئ ملف JSON داخل مجلد `backups/`. هذا المجلد غير مرفوع إلى GitHub حسب إعدادات `.gitignore`.

يمكن اختيار اسم أو مكان الملف:

```bash
npm run backup:learning -- --dir ../backups --file learning-content-latest.json
```

## فحص ملف قبل الاسترجاع

الاسترجاع يعمل افتراضيًا كفحص فقط ولا يكتب في قاعدة البيانات:

```bash
npm run restore:learning -- --file ../backups/learning-content-latest.json
```

## استرجاع آمن بدون حذف الحالي

هذا يحدث أو يضيف العناصر حسب `_id` ولا يحذف العناصر غير الموجودة في النسخة:

```bash
npm run restore:learning -- --file ../backups/learning-content-latest.json --apply
```

## استرجاع كامل مع استبدال مجموعات التعلم

استخدمه فقط عند الرجوع لنسخة قديمة بعد مشكلة كبيرة، لأنه يمسح مجموعات التعلم المدارة ثم يعيد بناءها من ملف النسخة:

```bash
npm run restore:learning -- --file ../backups/learning-content-latest.json --apply --replace
```

## ملاحظات أمان

- لا ترفع ملفات `backups/` إلى GitHub.
- احتفظ بنسخة خارج الجهاز: قرص خارجي أو Google Drive.
- قبل أي استرجاع حقيقي، شغل الأمر بدون `--apply` أولًا.
- الكود نفسه محفوظ على GitHub، أما هذا النسخ فهو لحماية محتوى MongoDB.
