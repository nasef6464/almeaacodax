# متابعة الدفعة الحالية

آخر تحديث: 2026-05-07

## الهدف

إغلاق كل جزئية بفحص واضح قبل الانتقال لغيرها، مع الحفاظ على بساطة واجهة الطالب وولي الأمر، وترك التفاصيل للإدارة.

## مغلق ولا يعاد فتحه إلا عند خلل

- مشغل الاختبار العادي وساهر: عداد الأسئلة، خريطة الأسئلة، حفظ التقدم، الإيقاف المؤقت، وتكبير صورة السؤال.
- صفحة النتيجة ومراجعة الحلول: ملخص بسيط أولًا، مراجعة بنفس أسئلة الاختبار، والتفاصيل بزر منفصل.
- رجوع الاختبار والتدريب إلى نفس سياق المادة عبر `returnTo` و`source`.

## الدفعة الحالية

مركز المكتبة وملفات الدعم.

تم تثبيت فحص `smoke:library-support` ليحرس الآتي:

- المدير يحفظ ملف المكتبة مربوطًا بالمسار والمادة والمهارة الرئيسية والمهارات الفرعية والرابط.
- ملف الدعم الجاهز للطالب لا يكون جاهزًا إلا إذا كان ظاهرًا، معتمدًا، له رابط، ومربوطًا بمهارة.
- الطالب لا يرى في المكتبة إلا الملفات المنشورة والمعتمدة داخل نفس المادة.
- ملف الدعم داخل موضوع التأسيس يظهر كزر/تبويب عملي، وليس بطاقة شرح مزعجة.
- ملف الدعم يفضل نفس المهارة الفرعية أولًا، ثم يرجع لملفات المهارة الرئيسية عند الحاجة.

## التالي بعد إغلاق هذه الدفعة

1. تنظيم صلاحيات الاختبارات المجانية والمدفوعة والباقات.
2. مركز الاختبارات المحاكية المستقل حسب المسار.
3. تقارير الطالب وولي الأمر والمشرف والمدير بشكل مختصر للطالب وولي الأمر، وتفصيلي للإدارة.
4. ساهر متعدد المهارات مع واجهة بسيطة.

## فحوص الدفعة

- `npm run smoke:quiz-access`
- `npm run smoke:library-support`
- `npm run smoke:learning-quiz`
- `npm run smoke:results`
- `npm run build`
- `npm --prefix server run build`

## الدفعة المفتوحة الآن

صلاحيات الاختبارات والباقات.

حارس الدفعة `smoke:quiz-access` يثبت أن الاختبار المجاني يفتح للطالب، والمدفوع/داخل الدورة يعتمد على باقة المادة، والموجه/الخاص لا يظهر إلا للمجموعة أو الطالب المحدد، وأن اختيار الظهور من إدارة مساحة التعلم هو الذي يتحكم فيما يراه الطالب في تبويب التدريب أو الاختبارات.
