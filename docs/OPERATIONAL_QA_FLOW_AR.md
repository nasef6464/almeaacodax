# دليل الفحص التشغيلي السريع

هذا المرجع يثبت ما يجب اختباره بعد كل دفعة تطوير، حتى نحافظ على الأجزاء التي ثبتت ولا نكسرها لاحقًا.

## 1. قاعدة السرية

- لا تكتب كلمات مرور أو أسرار أو مفاتيح API داخل المستندات.
- حسابات التشغيل التجريبية تراجع من مصدر آمن خارج Git.
- عند مشاركة تقرير فحص، اذكر الدور والنتيجة فقط بدون بيانات اعتماد.

## 2. فحص المدير

1. تسجيل الدخول بحساب مدير من مصدر آمن.
2. فتح لوحة الإدارة.
3. التأكد من ظهور النظرة العامة التشغيلية.
4. فتح إدارة المستخدمين والتأكد من ظهور الطلاب والمعلمين والمشرفين وأولياء الأمور.
5. فتح مركز الدروس والاختبارات والمكتبة والتأكد من ظهور المحتوى المعلق للمراجعة.
6. فتح مركز المهارات والتأكد أن الربط ما زال مبنيًا على:
   `المسار -> المادة -> المهارة الرئيسية -> المهارة الفرعية`.

## 3. فحص طلبات الدفع

1. فتح لوحة الإدارة ثم المالية والاشتراكات.
2. فتح تبويب طلبات الدفع.
3. التأكد من أن الفلاتر تعمل:
   - بانتظار المراجعة
   - كل الطلبات
   - معتمدة
   - مرفوضة
   - ملغية
4. البحث برقم الطلب أو البريد أو اسم الطالب أو مرجع الدفع.
5. التأكد أن الطلب الناقص يظهر عليه تنبيه تدقيق قبل الاعتماد.
6. اعتماد طلب واحد فقط عند الحاجة، وعدم اعتماد طلبات اختبارية على الإنتاج إلا بتنسيق واضح.

## 4. فحص المعلم

1. تسجيل الدخول بحساب معلم من مصدر آمن.
2. فتح لوحة العمل.
3. التأكد من ظهور المسار أو المادة المسندة فقط.
4. إضافة درس أو سؤال أو اختبار تجريبي عند الحاجة.
5. التأكد أن المحتوى الجديد يذهب للمراجعة ولا يظهر للطلاب مباشرة.

## 5. فحص المشرف

1. تسجيل الدخول بحساب مشرف من مصدر آمن.
2. فتح لوحة العمل.
3. التأكد من ظهور المدرسة أو المجموعة الخاصة به فقط.
4. مراجعة تقرير الطلاب الضعاف وتقرير المدرسة.
5. التأكد أن بيانات الطلاب خارج نطاقه لا تظهر.

## 6. فحص ولي الأمر

1. تسجيل الدخول بحساب ولي أمر من مصدر آمن.
2. فتح التقارير.
3. التأكد من ظهور الطالب المرتبط به فقط.
4. فتح تفاصيل نتيجة اختبار والتأكد من أن الملخص والمهارات والتوصيات باللغة العربية السليمة.

## 7. فحص الطالب

1. تسجيل الدخول بحساب طالب من مصدر آمن.
2. فتح الصفحات الأساسية:
   - الرئيسية
   - المسارات
   - الاختبارات
   - النتائج
   - التقارير
   - الخطة
3. التأكد من ظهور المحتوى المنشور فقط.
4. فتح مسار القدرات الكمي واللفظي والتحصيلي، ثم فتح مادة وموضوع ودرس.
5. التأكد أن الفيديو المرتبط بالدرس يعمل ولا تظهر بدائل مربكة للطالب.
6. فتح نتيجة اختبار والتأكد من:
   - ظهور النتيجة بوضوح.
   - ترتيب المهارات من الأضعف للأقوى.
   - عمل نسخ الملخص والمشاركة وPDF.

## 8. الفحص الآلي

يوجد فحص تشغيلي عبر:

```bash
npm run smoke:operational
```

هذا الفحص يتحقق من:

- تسجيل الدخول لكل دور أساسي.
- اتصال Render بقاعدة البيانات.
- إخفاء المحتوى المعلق عن الطالب والزائر.
- جاهزية مساحات التعلم الأساسية.
- وجود دروس قابلة للتشغيل داخل الموضوعات المنشورة.
- نطاق المعلم والمشرف وولي الأمر.
- إعدادات الدفع العامة والإدارية.

ولفحص واجهة Vercel:

```bash
npm run smoke:frontend
```

إذا ظهر تحذير `STALE` فهذا يعني أن Vercel ما زال يعرض نسخة قديمة. انتظر اكتمال النشر أو أعد نشر فرع `main`.

## 9. قواعد لا يجب كسرها

- المحتوى المعلق لا يظهر للطالب أو الزائر.
- واجهة الطالب تبقى بسيطة، والإحصاءات التشغيلية تبقى داخل الإدارة.
- صفحة تعلم المهارة تفتح المسار الموحد، لا صفحات قديمة.
- الباقات مفصولة حسب نوع المحتوى والمسار والمادة.
- طلبات الدفع لا تعتمد تلقائيًا إلا حسب سياسة الدفع المفعلة.
- لا يتم نشر أسرار أو كلمات مرور داخل Git.
